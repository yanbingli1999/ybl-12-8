# 星舰骰子指挥 - 战斗机制Bug调试

**Session ID**: `battle-mechanics-bugs`
**Status**: `[OPEN]`
**Created**: 2026-06-13
**Bugs**: 能量不扣、六点暴击无效、敌舰特殊意图放错、防御回滚异常

---

## 🔍 问题描述

### Bug 1: 确认回合后能量不扣
- **预期**: 分配骰子后应根据点数消耗能量
- **实际**: 能量值在确认回合后没有变化

### Bug 2: 六点暴击无效
- **预期**: 骰子值为6时应额外增加暴击率
- **实际**: 六点骰子没有暴击加成效果

### Bug 3: 敌舰特殊意图放错
- **预期**: 敌舰特殊意图应正确释放对应技能
- **实际**: 特殊技能释放逻辑错误，技能名称/ID不匹配

### Bug 4: 防御回滚异常
- **预期**: 敌方防御意图的加成在回合结束后正确回滚
- **实际**: 防御加成/减免计算顺序错误

---

## 🧪 可证伪假设

| ID | 假设 | 验证点 |
|----|------|--------|
| H1 | 能量消耗计算后Math.floor导致消耗为0 | `config.energyCostPerPoint`默认0.05，总点数需≥20才扣1能量 |
| H2 | 六点暴击逻辑正确但暴击率太低未被观察到 | 检查`bonusCritRate`计算和实际暴击触发情况 |
| H3 | `executeEnemyIntent`执行特殊技能时未按意图选择，随便选了冷却好的技能 | `executeEnemyIntent`用`find(a => a.currentCooldown === 0)`而非按意图中的技能名 |
| H4 | 防御加成时机错误，加在敌方攻击阶段而非玩家攻击阶段 | 防御+0.2在玩家攻击后才生效，完全没用 |
| H5 | `confirmTurn`中的敌方行动执行顺序错误，导致状态不一致 | 检查玩家回合→敌方回合的状态转换逻辑 |
